台灣知識庫

( 2013/1 ~ 2015/05 ) 資訊專員

在這裡我學到了網站應該要如何從零開始建置，從Tomcat、資料庫伺服器的架設，到網站的撰寫、測試，掛載至伺服器，最後再設定排程來保持伺服器的穩定性， 把這些流程全部實際熟悉、操作過後，我具備了自己架設網站的能力。除了網站的建置外，技術上的培養是更加的重要，後端方面我學習了Java的架構（Spring、Struts、Hibernate、Mybatis）， 並加強將程式元件化的能力，而由於接手了PHP的案子，對LAMP也有一定程度的熟悉，前端則是學會了JQuery、JavaScript、Ajax、Json等好用的技術，為網頁上的控制與效果打下了不少基礎， 由於公司網站眾多，接觸了不少不同種類的資料庫，包含Oracle、MySQL、PostgreSQL，也掌握了不少方便的語法如join、decode、when case等，才知道資料庫原來也可以做許多事情， 包含在package裡面寫程式，或是用procedure來做許多複雜的處理。

為了處理一些伺服器上的問題，我開始熟悉Windows批次檔的撰寫，不論是要自動開始、結束網站上的活動排程，或是重啟Tomcat、自動寄信（搭配Java程式）、監控CPU使用率、清除過期log，寫起來都相當有意思。 Linux伺服器也接觸了不少次，在Linux上架伺服器又是另外一種風味，用yum就可以解決大多數的流程，排程的設定則是使用crontab，由於大部分Linux的伺服器都有比較嚴格的權限控管， 必須熟悉chmod、chown指令來對檔案做權限和使用者的變更。

璦輝雲端資訊

( 2015/06 ~ 現今 ) 資訊專員

這間公司用了相當多以前沒有接觸過，對我來說是相當新的技術，像是後端的Groovy語言、Grails架構，用來編譯的Maven、Gradle， 寫前端的Ember.js，雲端的AWS服務，佈署網站用的Jenkins，資料庫Redis，還有做資料庫migration的相關技術如flyway、Grails Database Migration Plugin 都讓我大開眼界，另外利用Trello來做任務導向式的工作分配，自主的進度管理，都有相當高的靈活性， 使我對需求的規劃、時程的計算更有概念。

在這裡學到最重要的東西並不是技術，而是設計程式的觀念，一個功能、甚至是一整個專案，只要有重複利用的可能， 就應該要實現元件化：PayPal付款的功能，應該要能夠輕鬆的套用在其他的專案上；處理某專案用的表單系統，應該要設計成之後可以用在其他專案上， 這樣的觀念不僅能夠讓工作事半功倍，還能讓產出的程式更加優質、可讀。
